Combitech 2008

Master's thesis which investigated what sort of hard real-time requirements could be acchieved on a Linux based embedded system. Three different solutions where put to the test when their interrupt and scheduling latencies where measured using a custom external test tool. One solution, Xenomai, used a micro kernel approach, where all real-time tasks where run with a higher priority than the Linux kernel. Another, the PREEMPT_RT patch set, modified some fundamental kernel subsystems, tuning them for higher determinism. As a reference, an unmodified Linux kernel was also put through the test.

Measurements where performed under various degrees of load, and the results where presented in the report "Performance Evaluation of GNU/Linux for Real-Time Applications". The report is available at uu.diva-portal.org.
